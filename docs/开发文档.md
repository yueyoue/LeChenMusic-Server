# Navidrome 有声书扩展 — 开发文档

> 基于 Navidrome fork，参考 [ting-reader](https://github.com/dqsq2e2/ting-reader) 设计，为 Navidrome 增加完整的有声书功能。

---

## 1. 项目概述

### 1.1 目标

在 Navidrome（Go + React）基础上，扩展有声书模块，使其成为**音乐 + 有声书一体化**的个人流媒体平台。

### 1.2 技术栈

| 层 | 技术 | 说明 |
|---|---|---|
| 后端 | Go | 沿用 Navidrome |
| 前端 | React + Material-UI | 沿用 Navidrome Web UI |
| 数据库 | SQLite | 沿用 Navidrome（新增表） |
| API | Subsonic API + 自定义 REST | 扩展 Subsonic 协议 |
| 部署 | Docker | 沿用 Navidrome |

### 1.3 与 ting-reader 的关系

ting-reader 是 Rust + React（Vite + Tailwind + Zustand）架构，与 Navidrome（Go + React + Material-UI + Redux）**后端语言不同，前端框架不同**。不能直接复用代码，但可以参考：

- **UI 设计**：首页布局、书架展示、播放器交互、管理后台
- **功能设计**：刮削策略、进度同步机制、跳过片头片尾
- **数据模型**：书籍/章节/进度/书签结构

CREATE TABLE audiobook (
    id          VARCHAR(255) PRIMARY KEY,
    title       VARCHAR(255) NOT NULL,
    author      VARCHAR(255),          -- 作者
    narrator    VARCHAR(255),          -- 演播者（类似音乐的 Artist）
    description TEXT,                  -- 简介
    genre       VARCHAR(128),          -- 分类：有声读物/相声/评书/戏曲/儿童/教育
    year        INTEGER,

    chapter_count  INTEGER,            -- 章节数
    series      VARCHAR(255),          -- 系列名（如"鬼吹灯系列"）
    library_id  INTEGER REFERENCES library(id),
    path        VARCHAR(1024) NOT NULL, -- 有声书根目录（相对路径）
    hash        VARCHAR(64),           -- 目录哈希，用于变更检测
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 有声书章节（一个音频文件 = 一章）
CREATE TABLE audiobook_chapter (
    id            VARCHAR(255) PRIMARY KEY,
    audiobook_id  VARCHAR(255) REFERENCES audiobook(id) ON DELETE CASCADE,
    title         VARCHAR(255) NOT NULL,
    chapter_number INTEGER NOT NULL,
    duration      INTEGER,             -- 时长（秒）
    format        VARCHAR(20),         -- mp3/m4a/flac/...
    file_size     INTEGER,
    path          VARCHAR(1024) NOT NULL,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 播放进度（每用户每书一条记录）
CREATE TABLE audiobook_progress (
    id              VARCHAR(255) PRIMARY KEY,
    user_id         VARCHAR(255) REFERENCES user(id) ON DELETE CASCADE,
    audiobook_id    VARCHAR(255) REFERENCES audiobook(id) ON DELETE CASCADE,
    chapter_id      VARCHAR(255) REFERENCES audiobook_chapter(id),
    chapter_number  INTEGER,
    position        INTEGER DEFAULT 0,   -- 章节内播放位置（秒）
    playback_speed  REAL DEFAULT 1.0,    -- 播放速度
    skip_intro      INTEGER DEFAULT 0,   -- 跳过片头秒数
    skip_outro      INTEGER DEFAULT 0,   -- 跳过片尾秒数
    completed       BOOLEAN DEFAULT FALSE,
    last_played_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, audiobook_id)
);

-- 书签
CREATE TABLE audiobook_bookmark (
    id            VARCHAR(255) PRIMARY KEY,
    user_id       VARCHAR(255) REFERENCES user(id) ON DELETE CASCADE,
    audiobook_id  VARCHAR(255) REFERENCES audiobook(id) ON DELETE CASCADE,
    chapter_id    VARCHAR(255) REFERENCES audiobook_chapter(id),
    position      INTEGER NOT NULL,      -- 书签位置（秒）
    title         VARCHAR(255),          -- 书签备注
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- 有声书收藏（独立于音乐收藏，或复用 star 表加 type 字段）
CREATE TABLE audiobook_favorite (
    id            VARCHAR(255) PRIMARY KEY,
    user_id       VARCHAR(255) REFERENCES user(id) ON DELETE CASCADE,
    audiobook_id  VARCHAR(255) REFERENCES audiobook(id) ON DELETE CASCADE,
    created_at    DATETIME DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, audiobook_id)
);

### 3.2 Genre 自动识别

```go
var genreKeywords = map[string]string{
    "有声书":   "有声读物",
    "有声读物": "有声读物",
    "小说":    "有声读物",
    "评书":    "评书",
    "相声":    "相声",
    "戏曲":    "戏曲",
    "儿童":    "儿童",
    "教育":    "教育",
}

func detectGenre(relPath string) string {
    parts := strings.Split(filepath.ToSlash(relPath), "/")
    for i := 0; i < len(parts)-1; i++ {
        if genre, ok := genreKeywords[parts[i]]; ok {
            return genre
        }
    }
    return "有声读物" // 默认
}
```

### 3.3 变更检测

- `ChapterCount == 0` 的记录强制重新扫描

### 6.2 跳过片头片尾（参考 ting-reader）

```go
// 播放时应用跳过
func (p *Player) playChapter(chapter *AudiobookChapter, progress *AudiobookProgress) {
    p.load(chapter.StreamURL)
    
    if progress != nil && progress.Position > 0 {
        // 从上次位置继续
        p.seekTo(progress.Position * 1000)
    } else if progress != nil && progress.SkipIntro > 0 {
        // 跳过片头
        p.seekTo(progress.SkipIntro * 1000)
    }
    
    // 设置播放速度
    p.setSpeed(progress.PlaybackSpeed)
}

// 章节即将结束时，检测是否需要跳过片尾
func (p *Player) onChapterEnd() {
    if p.progress.SkipOutro > 0 {
        remaining := p.duration - p.position
        if remaining <= p.progress.SkipOutro*1000 {
            p.nextChapter() // 自动跳到下一章
        }
    }
}
```

### 6.3 自动刮削（参考 ting-reader）

```go
// scraper/metadata.go
type MetadataScraper interface {
    Scrape(ctx context.Context, bookPath string) (*AudiobookMetadata, error)
}

// 内置刮削源优先级：
// 1. 本地元数据（ID3 tags / folder.nfo / metadata.json）
// 2. 目录名解析（"作者 - 书名" / "书名 (作者)"）
// 3. 在线刮削（可选，需配置 API Key）

// 目录名解析规则
func parseDirName(name string) (author, title string) {
    // Pattern: "Author - Title"
    if idx := strings.Index(name, " - "); idx > 0 {
        return strings.TrimSpace(name[:idx]), strings.TrimSpace(name[idx+3:])
    }
    // Pattern: "Title (Author)"
    if start := strings.LastIndex(name, "("); start > 0 {
        if end := strings.LastIndex(name, ")"); end > start {
            return strings.TrimSpace(name[start+1 : end]), strings.TrimSpace(name[:start])
        }
    }
    // Default: entire name is title
    return "", name
}
```

const extractDominantColor = async (imageUrl) => {
  const canvas = document.createElement('canvas')
  const ctx = canvas.getContext('2d')
  const img = new Image()
  img.crossOrigin = 'anonymous'
  img.src = imageUrl
  await new Promise(r => img.onload = r)
  canvas.width = 1; canvas.height = 1
  ctx.drawImage(img, 0, 0, 1, 1)
  const [r, g, b] = ctx.getImageData(0, 0, 1, 1).data
  return `rgb(${r},${g},${b})`
}

### 6.5 收藏功能方案

**方案 A：独立收藏表（推荐）**

优点：与音乐收藏完全隔离，不影响现有逻辑。

```go
// 复用 Navidrome 的 star/unstar 模式，但使用独立表
func (r *audiobookRepo) Star(userID, bookID string) error {
    _, err := r.put(id.NewRandom(), &AudiobookFavorite{
        UserID: userID, AudiobookID: bookID,
    })
    return err
}
```

**方案 B：复用 star 表 + type 字段**

```sql
-- 在现有 star 表增加 type 字段
ALTER TABLE star ADD COLUMN type VARCHAR(20) DEFAULT 'song';
-- type: 'song' / 'album' / 'artist' / 'audiobook'
```

推荐方案 A，改动最小，风险最低。

## 9. 与 Navidrome 上游同步策略

### 9.1 原则

- **能新增文件就不改原有文件**
- 所有修改上游文件的地方加标记注释
- 定期从上游合并

### 9.2 标记规范

```go
// [LeChenMusic-START:audiobook] 有声书功能
// 合并上游时检查此区域
audiobookRepo := persistence.NewAudiobookRepository(ctx, db)
// [LeChenMusic-END:audiobook]
```

### 9.3 合并流程

```bash
git remote add upstream https://github.com/navidrome/navidrome.git
git fetch upstream
git merge upstream/master
# 解决冲突（只处理标记区域）
# 测试
git commit
```

## 11. 兼容性

### 11.1 Subsonic 客户端

通过扩展 Subsonic API，现有第三方客户端（如 Symfonium、Substreamer）可以：
- 浏览有声书列表
- 播放有声书章节
- 但不能使用高级功能（进度同步、书签等）

### 11.2 Web UI

Navidrome 现有的 React + Material-UI 框架完全支持新增页面，无需修改构建流程。

### 11.3 数据库

使用 Navidrome 的迁移系统，新增表不影响现有数据。
