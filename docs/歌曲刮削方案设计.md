# 歌曲刮削功能方案设计

## 1. 概述

为 WEB 端歌曲列表页增加元数据刮削功能，自动从在线音乐平台获取歌曲的封面、歌词、艺术家信息、专辑信息等元数据，并写入到本地音乐文件的 ID3 Tag 中。

## 2. 功能范围

### 2.1 刮削内容

| 字段 | 说明 | 来源 |
|------|------|------|
| 封面图 (Cover Art) | 专辑封面 | 网易云/QQ音乐/酷狗/酷我 |
| 歌词 (Lyrics) | LRC 格式歌词 | 网易云/QQ音乐 |
| 艺术家头像 (Artist Image) | 演唱者图片 | 已有（复用现有刮削） |
| 专辑信息 (Album Info) | 专辑名、年份、流派 | 网易云/QQ音乐 |
| 作词/作曲 (Composer/Lyricist) | 创作者信息 | 网易云/QQ音乐 |
| 标签 (Tags) | 分类标签 | 各平台 |

### 2.2 操作模式

1. **单首刮削**：右键菜单 → 刮削元数据
2. **批量刮削**：选中多首歌曲 → 批量刮削
3. **自动刮削**：扫描新文件时自动刮削（可选，需配置）

## 3. 技术架构

### 3.1 后端设计

#### 3.1.1 新增 Scraper 接口扩展

```go
// server/scraper/music_scraper.go

// MusicScraper 音乐元数据刮削器接口
type MusicScraper interface {
    Name() string
    DisplayName() string
    
    // SearchSong 搜索歌曲
    SearchSong(title, artist string) ([]SongSearchResult, error)
    
    // GetSongDetail 获取歌曲详情（含歌词、封面URL等）
    GetSongDetail(songID string) (*SongDetail, error)
    
    // GetLyrics 获取LRC格式歌词
    GetLyrics(songID string) (string, error)
    
    // GetAlbumCover 获取专辑封面URL
    GetAlbumCover(albumID string) (string, error)
}

type SongSearchResult struct {
    Source    string  `json:"source"`
    ID        string  `json:"id"`
    Title     string  `json:"title"`
    Artist    string  `json:"artist"`
    Album     string  `json:"album"`
    AlbumID   string  `json:"albumId"`
    Duration  float32 `json:"duration"`
    CoverURL  string  `json:"coverUrl"`
    HasLyrics bool    `json:"hasLyrics"`
}

type SongDetail struct {
    Source      string   `json:"source"`
    ID          string   `json:"id"`
    Title       string   `json:"title"`
    Artist      string   `json:"artist"`
    Album       string   `json:"album"`
    AlbumID     string   `json:"albumId"`
    CoverURL    string   `json:"coverUrl"`
    Lyrics      string   `json:"lyrics"`
    Year        int      `json:"year"`
    Genre       string   `json:"genre"`
    Composer    string   `json:"composer"`
    Lyricist    string   `json:"lyricist"`
    TrackNumber int      `json:"trackNumber"`
}
```

#### 3.1.4 元数据写入

刮削结果需要写入到：
1. **数据库**：更新 `media_file` 表的对应字段
2. **文件 ID3 Tag**：写入到音频文件的元数据标签中（可选，需配置）
3. **封面文件**：保存到歌曲所在目录的 `cover.jpg`

```go
// 元数据写入流程
func applySongScrape(ctx context.Context, songID string, detail *SongDetail) error {
    // 1. 更新数据库
    song, _ := repo.Get(songID)
    if detail.Title != "" { song.Title = detail.Title }
    if detail.Artist != "" { song.Artist = detail.Artist }
    // ... 其他字段
    repo.Put(song)
    
    // 2. 下载并保存封面
    if detail.CoverURL != "" {
        downloadAndSaveCover(detail.CoverURL, song.Path)
    }
    
    // 3. 保存歌词文件
    if detail.Lyrics != "" {
        saveLyricsFile(song.Path, detail.Lyrics)
    }
    
    // 4. 写入 ID3 Tag（可选）
    if conf.Server.AutoWriteID3Tags {
        writeID3Tags(song.Path, detail)
    }
}
```

## 4. 匹配算法

歌曲匹配是刮削准确性的关键：

```go
// matchScore 计算歌曲匹配分数
func matchScore(query *SongQuery, result *SongSearchResult) int {
    score := 0
    
    // 标题匹配（权重最高）
    if normalize(query.Title) == normalize(result.Title) {
        score += 100
    } else if contains(query.Title, result.Title) {
        score += 60
    }
    
    // 艺术家匹配
    if normalize(query.Artist) == normalize(result.Artist) {
        score += 50
    }
    
    // 专辑匹配
    if normalize(query.Album) == normalize(result.Album) {
        score += 30
    }
    
    // 时长匹配（±5秒）
    if math.Abs(query.Duration - result.Duration) < 5 {
        score += 20
    }
    
    return score
}
```

## 7. 风险与注意事项

1. **API 可用性**：各平台 API 可能变更，需要维护和更新
2. **反爬策略**：频繁请求可能被封 IP，需要限流和代理支持
3. **版权问题**：歌词等元数据的版权问题需要注意
4. **文件写入安全**：写入 ID3 Tag 需要备份原文件
5. **性能影响**：大量歌曲刮削时需要限流，避免影响正常播放服务
